THE 10 AI CAPABILITIES
================================================================================

1. SMART MATCH EXPLANATIONS
   Purpose: Explain why a developer is good/challenging for a project
   Example: "Sara's strong Python skills make her ideal for Budget Analytics.
            She'll need to learn database design (2-3 weeks)."
   Use in: Developer dashboard, match cards

2. CAREER DEVELOPMENT PATHS
   Purpose: Personalized career progression recommendations
   Example: "Learn Node.js next (4-6 weeks), then target Backend roles.
            Timeline to Mid-level: 12-18 months."
   Use in: Developer profile, career guidance section

3. SKILL GAP ANALYSIS
   Purpose: Identify missing skills and recommend training
   Example: "Missing Database Design. Free resources: freeCodeCamp (8h),
            Practice timeline: 3-4 weeks for junior developer."
   Use in: Project preparation, training planning

4. HIRING RECOMMENDATIONS
   Purpose: Strategic guidance on which developers to hire
   Example: "Priority order: Emily (92%), Michael (78%) + training,
            Sarah (65%) as growth investment."
   Use in: Employer dashboard, project staffing

5. SUCCESS PREDICTION
   Purpose: Predict likelihood of success for a match
   Example: "Success probability: 85%. Strong: has 3/4 skills.
            Challenge: new to DevOps (needs mentoring)."
   Use in: Risk assessment, match quality indication

6. WORKFORCE ANALYSIS
   Purpose: Organization-wide skills gap and hiring strategy
   Example: "Missing skills: Cloud Infrastructure, DevOps.
            Recommendation: Hire 1-2 specialists + train internally (3 months)."
   Use in: Strategic planning, hiring roadmap

7. JOB DESCRIPTIONS
   Purpose: AI-generated engaging project postings
   Example: "Transform how the City manages budgets with data analysis
            and scalable database design. Ideal for mid-level developers."
   Use in: Project descriptions, job postings

8. RESUME PARSING
   Purpose: Extract skills from resume text
   Example: Parse PDF/text resume → Auto-populate developer profile
   Use in: Quick onboarding, bulk hiring

9. TEAM COMPOSITION ANALYSIS
   Purpose: Analyze strengths/gaps of a team
   Example: "Strong backend (Emily) + learning curve (Sarah).
            Balance: experienced + growth mindset."
   Use in: Team assembly, project planning

10. ONBOARDING PLANS
    Purpose: First-month schedule for new developers
    Example: Week 1: Setup & orientation
             Week 2: Skill development
             Week 3-4: Full contribution
    Use in: Onboarding process, success planning

================================================================================
HOW TO USE IN CODE
================================================================================

In your JavaScript:

// 1. Initialize (done automatically if key saved)
if (claudeAI) {
  console.log("Claude AI is ready!");
}

// 2. Get a smart match explanation
const dev = DataManager.getDeveloperById(1);
const proj = DataManager.getProjectById(1);
const explanation = await claudeAI.generateMatchExplanation(dev, proj, 85);
console.log(explanation);

// 3. Generate career path
const careerPath = await claudeAI.generateCareerPath(dev);
console.log(careerPath);

// 4. Analyze skill gaps
const gaps = await claudeAI.analyzeSkillGaps(dev, proj);
console.log(gaps);

// 5. Get hiring recommendations
const matches = MatchingEngine.findDevelopersForProject(proj.id);
const insights = await claudeAI.generateHiringInsights(proj, matches);
console.log(insights);

All methods are documented in claude-ai.js with examples.
